# Goods - 物品管理插件

<PluginSwiper plugin-name="goods" />

📦 **Goods 物品管理** - 让每件物品都有它的故事

## 核心功能

### 🏠 仓库管理
- **多仓库支持**：创建和管理多个仓库，组织不同空间的物品
- **自定义图标**：为每个仓库设置专属图标和颜色
- **仓库图片**：可添加仓库实景图片，直观展示
- **仓库统计**：自动统计各仓库的物品数量和价值

### 📦 物品记录管理
- **层级结构**：支持多层级子物品，精细化管理
- **基础信息**：名称、购买日期、购买价格等
- **图片存储**：为物品添加图片，使用相对路径存储
- **标签系统**：为物品添加多个标签，便于分类和搜索

### 🏷️ 自定义字段
- **灵活定义**：为物品添加任意自定义字段（键值对）
- **字段类型**：支持文本、数字、日期等多种字段类型
- **个性化**：根据物品类型添加特有属性
  - 电子设备：序列号、保修期、品牌、型号
  - 衣物：尺码、颜色、材质
  - 书籍：作者、ISBN、阅读状态

### 📊 使用记录
- **使用历史**：记录物品的使用历史，跟踪使用场景
- **最后使用时间**：自动跟踪最后使用日期
- **使用频次统计**：基于使用记录分析使用频率
- **未使用提醒**：自动统计一个月未使用的物品

### 🔍 搜索与筛选
- **按仓库筛选**：支持筛选特定仓库的物品
- **按名称搜索**：快速搜索物品名称
- **多种排序**：按价格、最后使用时间排序
- **排序偏好**：为每个仓库保存独立的排序偏好

### 👁️ 双视图模式
- **网格视图**：卡片式展示，直观浏览
- **列表视图**：紧凑列表，信息密度高
- **响应式布局**：根据屏幕宽度自动调整

## 使用场景

### 家庭物品管理
- **家电管理**：记录家电信息、保修卡、说明书，归类到"客厅"、"厨房"等仓库
- **衣物管理**：整理衣柜，按季节或类型分类，跟踪穿着频次
- **玩具管理**：为孩子管理玩具，记录喜爱程度和使用频次
- **工具管理**：整理工具箱，快速找到所需工具

### 个人资产管理
- **数码设备**：管理手机、电脑、相机等设备，记录序列号和购买日期
- **收藏品管理**：管理邮票、硬币、手办等收藏，添加自定义字段记录来源和价值
- **书籍管理**：记录藏书，跟踪阅读进度和最后阅读时间
- **证件管理**：重要证件信息备份（不含敏感数据）

### 办公用品管理
- **办公设备**：管理打印机、投影仪等设备，添加到"办公室"仓库
- **耗材追踪**：记录办公耗材库存，在"耗材仓库"中管理
- **资产登记**：公司固定资产登记管理，记录购买价格和位置
- **领用记录**：记录办公用品的使用历史，跟踪借用情况

## 操作指南

### 创建仓库
1. 点击「仓库」Tab
2. 点击右下角「+」按钮
3. 输入仓库名称（如"客厅"、"书房"）
4. 选择图标和颜色
5. 可选：添加仓库图片
6. 点击保存

### 添加物品
1. 在仓库详情或所有物品页面点击「+」按钮
2. 选择所属仓库
3. 填写基本信息（名称、购买日期、价格）
4. 上传物品图片
5. 添加标签（支持多个）
6. 在「自定义字段」中添加特有属性
7. 在「使用记录」中添加使用历史
8. 在「子物品」中添加子物品（可选）
9. 保存物品

### 管理子物品
1. 打开物品详情
2. 进入「子物品」标签页
3. 点击「添加子物品」
4. 填写子物品信息
5. 子物品可继续添加子物品（支持多层级）
6. 父物品的总价格会自动包含子物品价格

### 使用记录
1. 打开物品详情
2. 进入「使用记录」标签页
3. 点击「添加使用记录」
4. 选择使用日期
5. 输入备注（可选）
6. 最后使用时间会自动更新

## 高级功能

### 搜索与筛选
- **按仓库筛选**：在「物品」视图中点击筛选按钮，选择特定仓库
- **按名称搜索**：点击搜索按钮，输入物品名称进行搜索
- **多种排序**：点击排序按钮，可选择按价格、按最后使用时间排序
- **排序偏好**：为每个仓库单独保存排序偏好，下次进入时自动应用

### 统计分析
- **物品总数统计**：卡片视图显示所有仓库的物品总数
- **总价值统计**：自动计算所有物品的累计价值
- **未使用物品**：统计一个月未使用的物品数量（红色警告显示）
- **价格累加**：子物品价格自动累加到父物品总价格

### 事件系统
- **物品添加事件**：当新物品添加时，系统会广播 `goods_item_added` 事件
- **物品删除事件**：当物品删除时，系统会广播 `goods_item_deleted` 事件
- **其他插件监听**：其他插件可以监听这些事件进行联动操作

### 数据存储
- **本地存储**：所有数据存储在本地应用目录
- **图片路径**：使用相对路径存储图片，便于跨设备同步
- **路径转换**：自动处理相对路径与绝对路径的转换
- **数据持久化**：用户偏好、排序方式等自动保存

## 常见问题

**Q: 如何快速找到一个物品？**
A: 在「物品」视图中使用搜索功能，或按仓库筛选缩小范围。

**Q: 子物品的价格如何计算？**
A: 父物品的总价格会自动累加所有子物品的价格，支持多层级嵌套。

**Q: 图片存储在哪里？**
A: 图片存储在应用目录的 goods/goods_images/ 或 goods/warehouse_images/ 目录下。

**Q: 如何删除一个仓库？**
A: 在仓库详情页点击更多按钮，选择「删除仓库」，删除前会提示是否清空物品。

**Q: 可以导出物品数据吗？**
A: 当前版本暂无导出功能，数据存储在本地，建议通过 WebDAV 同步备份。

**Q: 如何批量管理物品？**
A: 目前需要逐个编辑物品，建议合理使用标签和仓库分类进行管理。

**Q: 物品 ID 是如何生成的？**
A: 使用 UUID v4 算法生成唯一标识符，确保每个物品的 ID 唯一。
